## Module 1: နိဒါန်းနှင့် လမ်းကြောင်းများ၏ သဘောတရား (The Nature of Paths)

ဒီ Module မှာတော့ Python ကို အသုံးပြုပြီး ဖိုင်စနစ် (File System) နဲ့ အလုပ်လုပ်တဲ့အခါ သိထားသင့်တဲ့ အခြေခံ သဘောတရားတွေ၊ အထူးသဖြင့် "Path" လို့ခေါ်တဲ့ လမ်းကြောင်းတွေအကြောင်းကို အဓိကထားပြီး နားလည်အောင် ရှင်းပြပေးသွားမှာပါ။

---

### 1.1 File System အလုပ်လုပ်ပုံ (File System and OS Differences)



မင်္ဂလာပါ ခင်ဗျာ။ ကျွန်တော်တို့ Python နဲ့ ဖိုင်တွေကို စီမံခန့်ခွဲတော့မယ်ဆိုရင်၊ ကိုယ်အလုပ်လုပ်နေတဲ့ ကွန်ပျူတာရဲ့ **ဖိုင်စနစ် (File System)** ကို အရင်ဆုံး နားလည်ထားဖို့ လိုအပ်ပါတယ်။ ဖိုင်စနစ်ဆိုတာကတော့ ကွန်ပျူတာထဲမှာ ဒေတာတွေကို စနစ်တကျ သိမ်းဆည်းထားတဲ့ ပုံစံပါပဲ။

ဖိုင်စနစ်မှာ အဓိက အစိတ်အပိုင်း (၂) ခုပဲ ရှိပါတယ်။

1.  **Directory (ဖိုဒါ):** အခြားဖိုင်များနှင့် ဖိုဒါများကို စုစည်းထားသော နေရာ။
2.  **File (ဖိုင်):** အမှန်တကယ် ဒေတာ (စာသား၊ ရုပ်ပုံ၊ ဗီဒီယို စသည်) များကို သိမ်းဆည်းထားသော အရာ။

ဒီဖိုင်တွေနဲ့ ဖိုဒါတွေကို သွားလာဖို့အတွက် **လမ်းကြောင်း (Path)** လိုအပ်ပါတယ်။ ဒီလမ်းကြောင်းတွေဟာ ကျွန်တော်တို့ သုံးတဲ့ **Operating System (OS)** ပေါ်မူတည်ပြီး ပုံစံကွဲပြားပါတယ်။ ဒါက Python Programmer တစ်ယောက်အနေနဲ့ အရေးကြီးဆုံး သိထားရမယ့် အချက်ပါ။

#### အဓိက OS များအကြား Path ပုံစံ ကွဲပြားပုံ

| OS အမျိုးအစား | Root Directory | Path Separator (လမ်းခွဲသင်္ကေတ) | ဥပမာ |
| :--- | :--- | :--- | :--- |
| **Windows** | Drive Letter (C:\, D:\) | Backslash (`\`) | `C:\Users\John\Documents\report.txt` |
| **Linux/macOS** | Root ( / ) | Forward Slash (`/`) | `/home/john/documents/report.txt` |

<Notice type="warning">
**သတိပြုရန်:** Python Code တွေ ရေးတဲ့အခါ Windows ရဲ့ Backslash (`\`) ကို တိုက်ရိုက် သုံးစွဲခြင်းက ရှောင်ရှားသင့်ပါတယ်။ ဘာကြောင့်လဲဆိုတော့ Backslash ဟာ Python မှာ Escape Character (ဥပမာ- <code>\n</code> က Newline ဖြစ်သွားတယ်) အဖြစ် သတ်မှတ်ထားလို့ပါ။ ဒါကြောင့် Python ဟာ OS အားလုံးမှာ အဆင်ပြေစေဖို့ **Forward Slash (`/`)** ကို အသုံးပြုဖို့ အားပေးပါတယ်။ နောက်လာမယ့် <code>pathlib</code> က ဒီပြဿနာကို လုံးဝ ဖြေရှင်းပေးပါလိမ့်မယ်။
</Notice>

### 1.2 Absolute vs Relative Paths (လမ်းကြောင်းအမျိုးအစားများ)

Path တွေကို အဓိကအားဖြင့် အမျိုးအစား (၂) မျိုး ခွဲခြားနိုင်ပါတယ်။



#### ၁။ Absolute Path (အပြည့်အစုံ လမ်းကြောင်း)

**Absolute Path** ဆိုတာက ဖိုင်စနစ်ရဲ့ အရင်းမြစ် (Root - Windows မှာ Drive Letter၊ Linux/macOS မှာ `/`) ကနေ စတင်ပြီး၊ သက်ဆိုင်ရာ ဖိုင် (သို့မဟုတ်) ဖိုဒါဆီကို အဆုံးထိ အတိအကျညွှန်ပြတဲ့ လမ်းကြောင်းပါ။

**အင်္ဂါရပ်များ:**

*   ဘယ်နေရာကနေ ခေါ်ခေါ် ဒီလမ်းကြောင်းက အမြဲတမ်း တူညီပါတယ်။
*   ဖိုင်စနစ်မှာ သူ့ရဲ့ တည်နေရာကို **လုံးဝ အတိအကျ** ဖော်ပြပေးပါတယ်။

```python
# Absolute Path ဥပမာများ
# Windows မှာ
windows_path = "C:/Users/John/data/file.txt"

# Linux/macOS မှာ
linux_path = "/Users/john/data/file.txt"
```

#### ၂။ Relative Path (ဆွေမျိုး လမ်းကြောင်း)

**Relative Path** ဆိုတာကတော့ ကျွန်တော်တို့ရဲ့ Python Script ဒါမှမဟုတ် Program ဟာ လက်ရှိ လုပ်ဆောင်နေတဲ့ နေရာ (Current Working Directory - CWD) ကို အခြေခံပြီး တွက်ချက်ထားတဲ့ လမ်းကြောင်းပါ။

**အင်္ဂါရပ်များ:**

*   CWD ပေါ် မူတည်ပြီး လမ်းကြောင်းပြောင်းလဲနိုင်ပါတယ်။
*   ဖိုင်စနစ်ရဲ့ တစ်စိတ်တစ်ပိုင်းကိုပဲ ညွှန်ပြပါတယ်။

Relative Path မှာ အရေးကြီးတဲ့ သင်္ကေတ နှစ်ခုကို သိထားဖို့ လိုအပ်ပါတယ်။

| သင်္ကေတ | အဓိပ္ပာယ် | ဥပမာ |
| :--- | :--- | :--- |
| **`.`** | လက်ရှိ လုပ်ဆောင်နေသော Directory (CWD) ကို ရည်ညွှန်းသည်။ | `./data/config.json` ဆိုသည်မှာ လက်ရှိဖိုဒါထဲမှ `data` ဖိုဒါအတွင်းရှိ ဖိုင်ကို ညွှန်းသည်။ |
| **`..`** | လက်ရှိ Directory ၏ အထက်/အပြင်ဘက် တစ်ဆင့်တက်သော Parent Directory ကို ရည်ညွှန်းသည်။ | `../images/photo.jpg` ဆိုသည်မှာ လက်ရှိဖိုဒါ၏ အပြင်ဖက်ဖိုဒါမှ `images` အတွင်းရှိ ဖိုင်ကို ညွှန်းသည်။ |

```python
# Relative Path ဥပမာများ
data_file = "config/settings.yaml" # CWD ထဲက config ထဲမှာရှိတဲ့ settings.yaml
parent_folder_data = "../backup/old_data.zip" # CWD ရဲ့ အပေါ်ဖိုဒါက backup ထဲမှာရှိတဲ့ data
```

<Notice type="info">
**Hands-On Tip:** Project သေးသေးလေးတွေ ရေးတဲ့အခါ၊ ကိုယ့် Project Folder ထဲက ဖိုင်တွေကို ကိုင်တွယ်ဖို့ **Relative Path** ကို သုံးတာက ပိုမို အဆင်ပြေလေ့ရှိပါတယ်။
</Notice>

### 1.3 Python Standard Library ၏ အရေးပါပုံ (`os` vs `pathlib`)



Python မှာ ဖိုင်စနစ်နဲ့ အလုပ်လုပ်ဖို့အတွက် အဓိက မော်ဂျူး (၂) ခုကို အသုံးပြုကြပါတယ်။ နှစ်ခုလုံးဟာ Python ရဲ့ **Standard Library** ထဲမှာ ပါဝင်ပါတယ်။ ဆိုလိုတာက ဒီ Module တွေကို သုံးဖို့အတွက် `pip install` လုပ်စရာ မလိုပါဘူး။

#### ၁။ `os` Module (The Old School / Traditional Approach)

`os` (Operating System) Module ဟာ Python ရဲ့ ရှေးအကျဆုံးနှင့် အစွမ်းထက်ဆုံး ဖိုင်စနစ် ကိုင်တွယ်ရေး Module တစ်ခုပါ။

*   **သဘောတရား:** Function-based ဖြစ်ပြီး၊ ဖိုင်စနစ်နဲ့ ပတ်သက်တဲ့ နိမ့်ကျတဲ့ (low-level) အလုပ်တွေကို လုပ်ဆောင်နိုင်ပါတယ်။
*   **အားနည်းချက်:**
    *   Path တွေကို String (စာသား) အဖြစ်ပဲ ကိုင်တွယ်ရတာကြောင့် မှားယွင်းမှု ဖြစ်လွယ်ပါတယ်။
    *   `os.path.join()` လိုမျိုး Function ကို သုံးပြီး OS အမျိုးအစားမတူတာကို ဖြေရှင်းပေးရပါတယ်။

```python
import os

# os ကို အသုံးပြုပြီး path တွေ ပေါင်းစပ်ပုံ
base_dir = "data"
file_name = "report.txt"

# os.path.join() ကို သုံးရတာက ရှုပ်ထွေးနိုင်ပါတယ်
# ဒါပေမယ့် OS မတူတာကို သူက ဖြေရှင်းပေးပါတယ်။
full_path_os = os.path.join(base_dir, file_name)

print(f"OS Path: {full_path_os}")
# (Windows မှာ 'data\\report.txt' လို့ ထွက်နိုင်ပြီး၊ Linux မှာ 'data/report.txt' လို့ ထွက်နိုင်ပါတယ်)
```

#### ၂။ `pathlib` Module (The Modern / OOP Approach)

`pathlib` Module ကိုတော့ Python 3.4 နောက်ပိုင်းမှာ စတင်မိတ်ဆက်ခဲ့တာဖြစ်ပြီး၊ ၎င်းသည် ခေတ်မီပြီး Object-Oriented (OOP) ပုံစံဖြင့် Path များကို ကိုင်တွယ်စေပါတယ်။

*   **သဘောတရား:** Path များကို String အစား **Path Object** အနေနဲ့ ကိုင်တွယ်ပါတယ်။
*   **အားသာချက်:**
    *   **Cross-Platform:** OS မတူညီမှုကို ဂရုစိုက်စရာ မလိုဘဲ Forward Slash (`/`) ကို သုံးပြီး Path တွေ ပေါင်းလို့ရပါတယ်။
    *   Code ကို ဖတ်ရလွယ်ကူပြီး၊ Path Object မှာ Functions တွေ ပါပြီးသား ဖြစ်ပါတယ်။

```python
from pathlib import Path

# Path Object ကို ဖန်တီးခြင်း
base_dir_path = Path("data")
file_name = "report.txt"

# Forward Slash / operator ကို သုံးပြီး လွယ်ကူစွာ ပေါင်းစပ်ခြင်း
full_path_pathlib = base_dir_path / file_name

print(f"Pathlib Path: {full_path_pathlib}")
# (OS ဘယ်လိုပဲဖြစ်ဖြစ် သင့်တော်တဲ့ Path ပုံစံကို အလိုအလျောက် ပြောင်းပေးပါလိမ့်မယ်။)
```

<Notice type="success">
**အနှစ်ချုပ်:** ကျွန်တော်တို့ဟာ `os` ရဲ့ အခြေခံ သဘောတရားတွေကို အရင်လေ့လာပြီး၊ ပိုမိုလွယ်ကူတဲ့၊ ခေတ်မီတဲ့ **`pathlib`** ကို အဓိကထားပြီး ဒီသင်တန်းတစ်လျှောက်လုံး အသုံးပြုသွားမှာ ဖြစ်ပါတယ်။
</Notice>
