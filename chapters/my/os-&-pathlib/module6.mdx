## Module 6: Main Project အကောင်အထည်ဖော်ခြင်း (Project Implementation)

**Main Project အမည်:** Desktop Folder Cleanup and Organization Tool

**ရည်ရွယ်ချက်:** သတ်မှတ်ထားသော Source Folder (ဥပမာ: Downloads Folder) အတွင်းရှိ ရှုပ်ထွေးနေသော ဖိုင်များကို အမျိုးအစားအလိုက် Destination Folder များသို့ အလိုအလျောက် ရွှေ့ပြောင်း စီမံခန့်ခွဲပေးမည့် Python Script ကို ရေးသားခြင်း။

---

### 6.1 Project Plan နှင့် Folder Structure (လုပ်ငန်းစဉ် အဆင့်ဆင့်)



ကောင်းမွန်တဲ့ Script တစ်ခုဖြစ်ဖို့အတွက် စနစ်တကျ စီစဉ်ဖို့ လိုပါတယ်။ ကျွန်တော်တို့ရဲ့ Cleanup Tool က အောက်ပါ အဆင့် (၄) ဆင့်အတိုင်း အလုပ်လုပ်မှာ ဖြစ်ပါတယ်။

1.  **Source & Destination သတ်မှတ်ခြင်း:** မည်သည့် Folder ကို ရှင်းလင်းမည်၊ မည်သည့် Folder များအတွင်းသို့ ရွှေ့မည်ကို သတ်မှတ်ခြင်း။
2.  **Extension Mapping Logic:** ဖိုင်အမျိုးအစား (Extension) ကို မည်သည့် Folder အောက်သို့ ပို့မည်ကို သတ်မှတ်ခြင်း။
3.  **ဖိုင်များ လှည့်ပတ်ရှာဖွေခြင်း:** Source Folder အတွင်းရှိ ဖိုင်များကို တစ်ခုချင်းစီ စစ်ဆေးခြင်း။
4.  **ရွှေ့ပြောင်းခြင်းနှင့် Error Handling:** တွေ့ရှိသော ဖိုင်များကို သက်ဆိုင်ရာ Destination Folder သို့ ရွှေ့ပြောင်းပြီး Error များကို စီမံခန့်ခွဲခြင်း။

#### ပရောဂျက်အတွက် အသုံးပြုမည့် Path များနှင့် Setup

```python
from pathlib import Path
import shutil
import sys # User Input အတွက် အသုံးပြုနိုင်

# ၁။ Source Folder ကို သတ်မှတ်ခြင်း (ဥပမာ- Desktop ကို ညွှန်းနိုင်)
# သင်ခန်းစာအတွက် 'Test_Junk' ဆိုတဲ့ Folder ကို CWD အောက်မှာ သုံးပါမယ်။
SOURCE_DIR = Path.cwd() / "Test_Junk"

# Destination Folder (စီမံခန့်ခွဲပြီးသား ဖိုင်တွေထားမယ့် နေရာ)
DEST_DIR = Path.cwd() / "Organized_Files"

# Folder တွေ မရှိရင် ဖန်တီးပါ
SOURCE_DIR.mkdir(exist_ok=True)
DEST_DIR.mkdir(exist_ok=True)
```

### 6.2 Extension Mapping Logic (ဖိုင်အမျိုးအစား ခွဲခြားခြင်း)



ဖိုင်တစ်ခုရဲ့ Extension ကိုကြည့်ပြီး ဘယ် Destination Folder ထဲကို ရွှေ့ရမယ်ဆိုတာကို သတ်မှတ်ဖို့ လိုပါတယ်။ ဒီအတွက် Python Dictionary ကို အသုံးပြုပါမယ်။

**၁။ Mapping Dictionary ဖန်တီးခြင်း**

```python
# Key: File Extension (Lowercase), Value: Destination Folder Name
FILE_MAPPINGS = {
    # Documents
    '.pdf': 'Documents/PDFs',
    '.docx': 'Documents/Word',
    '.xlsx': 'Documents/Excel',
    '.txt': 'Documents/Texts',
    
    # Images
    '.jpg': 'Images/Photos',
    '.png': 'Images/Photos',
    '.gif': 'Images/Photos',
    
    # Executables/Archives
    '.exe': 'Programs',
    '.zip': 'Archives',
    '.rar': 'Archives',
}

# Mapping ထဲမှာ မပါဝင်တဲ့ ဖိုင်တွေကို 'Others' folder ထဲ ပို့ပါမယ်
DEFAULT_FOLDER = 'Others'
```

**၂။ Extension ကို ထုတ်ယူပြီး Destination ကို ရှာဖွေခြင်း**

Module 3 မှာ လေ့လာခဲ့တဲ့ **`pathlib.Path.suffix`** ကို အသုံးပြုပြီး ဖိုင်ရဲ့ extension ကို အလွယ်တကူ ရယူနိုင်ပါတယ်။

```python
def get_destination_path(file_path: Path) -> Path:
    """ပေးထားသော Path အတွက် ရွှေ့ပြောင်းမည့် Destination Path ကို တွက်ချက်ပေးသည်"""
    
    # Extension ကို ရယူပြီး စာလုံးသေးအဖြစ် ပြောင်းပါ (e.g. .JPG -> .jpg)
    ext = file_path.suffix.lower()
    
    # FILE_MAPPINGS ထဲမှာ ရှိမရှိ စစ်ဆေးပါ
    subfolder_name = FILE_MAPPINGS.get(ext)
    
    if subfolder_name:
        # Mapping ထဲမှာ ပါရင် သက်ဆိုင်ရာ Subfolder ထဲပို့မယ်
        return DEST_DIR / subfolder_name
    else:
        # မပါရင် Others folder ထဲ ပို့မယ်
        return DEST_DIR / DEFAULT_FOLDER
```

### 6.3 Robust Script ရေးသားခြင်းနှင့် Error Handling (အမှားများကို စီမံခန့်ခွဲခြင်း)



ဖိုင်တွေကို ရွှေ့ပြောင်းတဲ့အခါ ဖြစ်ပေါ်လာနိုင်တဲ့ အမှားတွေ (ဥပမာ - Destination မှာ ဖိုင်အမည် တူနေတာ၊ Permission မရှိတာ) ကို `try...except` ဖြင့် ကိုင်တွယ်ဖို့ လိုအပ်ပါတယ်။

**၁။ Main Cleanup Logic ရေးသားခြင်း**

ဖိုင်တစ်ခုချင်းစီကို လှည့်ပတ်ပြီး ရွှေ့ပြောင်းမယ့် အဆင့်ပါ။

```python
print(f"\n--- {SOURCE_DIR} မှ ဖိုင်များ စီမံခန့်ခွဲမှု စတင်ပါပြီ ---")
file_count = 0

# Source Folder အတွင်းရှိ Item အားလုံးကို စစ်ဆေးမယ် (iterdir() ကို သုံး၍)
for item_path in SOURCE_DIR.iterdir():
    
    # Folder များကို ကျော်သွားပါ (ဖိုင်များသာ စီမံခန့်ခွဲမည်)
    if item_path.is_dir():
        continue
        
    # ဖိုင်ကို ရွှေ့မယ့် Destination Folder ကို တွက်ချက်ခြင်း
    destination_folder = get_destination_path(item_path)
    
    # Destination Folder မရှိသေးရင် ဖန်တီးပါ (mkdir(parents=True) က Nested Folder ကို ဖန်တီးပေးသည်)
    destination_folder.mkdir(parents=True, exist_ok=True)
    
    # ရွှေ့ပြောင်းမည့် Final Path
    final_dest_path = destination_folder / item_path.name
    
    try:
        # ဖိုင်ကို ရွှေ့ပြောင်းခြင်း (Path.replace() ကို သုံး၍)
        # replace() သည် destination မှာ ရှိနေပြီးသားဆိုလျှင် overwrite လုပ်ပါမည်။
        item_path.replace(final_dest_path)
        print(f"  [SUCCESS] '{item_path.name}' ကို '{destination_folder.name}' သို့ ရွှေ့ပြီး။")
        file_count += 1
        
    except Exception as e:
        # Permission Error ဒါမှမဟုတ် အခြား unexpected error များကို ကိုင်တွယ်ခြင်း
        print(f"  [ERROR] '{item_path.name}' ကို ရွှေ့ပြောင်းရာတွင် အမှားဖြစ်: {e}")

print(f"\n--- စီမံခန့်ခွဲမှု ပြီးဆုံးပါပြီ။ စုစုပေါင်း {file_count} ခု ရွှေ့ပြောင်းပြီး။ ---")
```

**၂။ Project ပြီးမြောက်မှု (Final Project အတွက် ပြင်ဆင်ခြင်း)**

ဒီ Script ဟာ `os` (Folder ဖန်တီးတဲ့အခါ `exist_ok` လို logic တွေ)၊ `pathlib` (Path Object တွေနဲ့ ကိုင်တွယ်တာ၊ `suffix` တွေ ယူတာ) နဲ့ `shutil` (မပါဝင်ပေမယ့် `replace()` က move လုပ်ပေးလို့) ရဲ့ အခြေခံ သဘောတရားတွေကို ပေါင်းစပ်အသုံးပြုသွားတာ ဖြစ်ပါတယ်။
